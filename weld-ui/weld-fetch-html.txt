      <h1>weld.fetch</h1>
      <span class="ver">v1.2.0</span>
  </div>
  <p class="lead">Outbound HTTP from a generator, done right: get/post/json helpers over the superFetch proxy, with timeouts, retries, normalized responses, and automatic direct routing for CORS-friendly hosts.</p>
</div>
<h2>Install</h2>
<p>Import weld.fetch alongside the superFetch plugin (so it can reach the proxy), and attach it. Stashes on <code>window.weld.fetch</code>. For CORS-friendly hosts it can also use the browser's own fetch, so superFetch is technically optional but recommended.</p>
<pre data-codeid="install"></pre>

<h2>Why you need this</h2>
<p>A generator runs in a sandboxed iframe, so a plain <code>fetch()</code> to a third-party API is usually blocked by CORS. Perchance routes outbound requests through the <strong>superFetch proxy</strong> (<code>fetch-plugin.perchance.org</code>). weld.fetch wraps that with real ergonomics and encodes the proxy's quirks so they don't surprise you:</p>
<ul>
<li>The proxy <strong>strips request headers</strong> &mdash; don't rely on sending custom ones.</li>
<li>Internal/SSRF targets are <strong>blocked</strong>.</li>
<li>Redirects are followed; there's <strong>no small body cap</strong>.</li>
<li>Identical URLs may be <strong>cached</strong> by the proxy.</li>
</ul>

<h2>json / text / get / post</h2>
<div class="sig" data-codeid="sig_json"></div>
<div class="sig" data-codeid="sig_verbs"></div>
<p>The one-step helpers cover the common cases: <code>json()</code> fetches and parses (throwing on non-2xx or non-JSON), <code>text()</code> returns the body string. <code>get()</code>/<code>post()</code>/<code>del()</code> return a normalized response.</p>
<pre data-codeid="json_ex"></pre>

<h2>request(url, opts)</h2>
<div class="sig" data-codeid="sig_req"></div>
<table>
<tr><th>opt</th><th>default</th><th>meaning</th></tr>
<tr><td><code>method</code></td><td>'GET'</td><td>HTTP method</td></tr>
<tr><td><code>body</code> / <code>json</code></td><td>—</td><td>request body; with <code>json:true</code> an object is stringified</td></tr>
<tr><td><code>direct</code></td><td>auto</td><td>force browser fetch (true) or proxy (false); auto-detects known CORS hosts</td></tr>
<tr><td><code>timeoutMs</code></td><td>15000</td><td>reject if no response in time</td></tr>
<tr><td><code>retries</code> / <code>retryDelayMs</code></td><td>0 / 600</td><td>retry with linear backoff</td></tr>
</table>
<p>It resolves a <strong>normalized</strong> response &mdash; <code>\&#123; ok, status, url, via, headers, text(), json(), raw \&#125;</code> &mdash; so you never juggle the proxy's boxed quirks. <code>via</code> is <code>'proxy'</code> or <code>'direct'</code>.</p>

<h2>Direct hosts</h2>
<div class="sig" data-codeid="sig_direct"></div>
<p>Hosts that already serve permissive CORS (raw.githubusercontent.com, cdn.jsdelivr.net, api.github.com) are hit directly, skipping the proxy hop. Add your own with <code>addDirectHost(hostname)</code>, or force per-request with <code>\&#123; direct: true \&#125;</code>.</p>
<div class="note">Pair with <code>weld.cache</code> to memoize responses (the proxy is a network hop worth saving), and with <code>weld.queue</code> if you need to rate-limit a burst of calls.</div>
<h2>Restricted mode (?$csp) <span class="tag">v1.1.0</span></h2>
<div class="sig" data-codeid="sig_csp"></div>
<p>Perchance&rsquo;s <code>?$csp</code> URL mode restricts a page from reaching external servers. Under it, both the superFetch proxy and direct fetches to outside hosts may be blocked by the platform. weld.fetch can&rsquo;t lift that, but <code>cspActive()</code> reports it, and a request that fails while it&rsquo;s in effect is annotated with that context &mdash; so a blocked call reads as &ldquo;CSP restricted this&rdquo; instead of a mysterious network error. Check it up front to disable outbound features gracefully.</p>

<script>
(function(){
'use strict';

if (!window.weld) window.weld = {};
if (window.weld.fetch) return window.weld.fetch;

// -----------------------------------------------------------------
// weld.fetch: reach the outside world from inside the sandbox.
//
// A generator runs in a sandboxed iframe, so a plain fetch() to a third
// party is usually blocked by CORS. Perchance provides superFetch, which
// routes requests through fetch-plugin.perchance.org. weld.fetch wraps it
// with the helpers people actually want and encodes the proxy's quirks:
//
//   - request headers are STRIPPED by the proxy (don't rely on them)
//   - internal/SSRF targets are blocked
//   - redirects are followed; there is no small body cap
//   - identical URLs may be cached by the proxy
//
//   await weld.fetch.json('https://api.example.com/data')   // GET + parse
//   await weld.fetch.post(url, { body: payload, json: true })
//   await weld.fetch.get(url, { timeoutMs: 8000, retries: 2 })
//
// For hosts that already send permissive CORS (raw.githubusercontent,
// jsdelivr, ...), set { direct: true } (or call when superFetch is
// absent) to use the browser's own fetch and skip the proxy hop.
// -----------------------------------------------------------------

var deps = { superFetch: null };
function getSF() {
  return deps.superFetch
    || (window.weld && window.weld.__superFetch)
    || (typeof window !== 'undefined' && typeof window.superFetch === 'function' ? window.superFetch : null)
    || (typeof superFetch === 'function' ? superFetch : null);
}
function attach(d) { if (d && d.superFetch) deps.superFetch = d.superFetch; }

// Hosts known to serve permissive CORS -- safe to hit directly, skipping
// the proxy. Extendable via opts.direct or addDirectHost().
var DIRECT_HOSTS = ['raw.githubusercontent.com', 'cdn.jsdelivr.net', 'api.github.com'];
function isDirectHost(url) {
  try { var h = new URL(url).hostname; return DIRECT_HOSTS.indexOf(h) !== -1; }
  catch (e) { return false; }
}
function addDirectHost(h) { if (h && DIRECT_HOSTS.indexOf(h) === -1) DIRECT_HOSTS.push(String(h)); }

function withTimeout(promise, ms, label) {
  if (!ms) return promise;
  return new Promise(function (resolve, reject) {
    var done = false;
    var t = setTimeout(function () { if (!done) { done = true; reject(new Error('weld.fetch: timeout after ' + ms + 'ms' + (label ? ' (' + label + ')' : ''))); } }, ms);
    Promise.resolve(promise).then(
      function (v) { if (!done) { done = true; clearTimeout(t); resolve(v); } },
      function (e) { if (!done) { done = true; clearTimeout(t); reject(e); } });
  });
}

// Core request. opts: { method, headers, body, json, direct, timeoutMs,
// retries, retryDelayMs }. Resolves a normalized response, NOT a raw
// Response, so callers don't have to remember .text()/.json() or the
// proxy's boxed quirks.
function request(url, opts) {
  opts = opts || {};
  url = String(url || '');
  if (!url) return Promise.reject(new Error('weld.fetch: empty URL'));

  var method = (opts.method || 'GET').toUpperCase();
  var direct = opts.direct === true || (opts.direct !== false && isDirectHost(url));
  var sf = getSF();
  if (!sf && !direct) {
    // No proxy available and not a known-direct host: try direct anyway,
    // but warn via the error if CORS then blocks it.
    direct = true;
  }
  var fetcher = direct ? (typeof fetch === 'function' ? fetch : null) : sf;
  if (!fetcher) return Promise.reject(new Error('weld.fetch: no fetch available (superFetch missing and no global fetch)'));

  var init = { method: method };
  if (opts.headers) init.headers = opts.headers; // note: proxy strips these
  if (opts.body != null) {
    init.body = (opts.json && typeof opts.body !== 'string') ? JSON.stringify(opts.body) : opts.body;
    if (opts.json && !direct) { init.headers = Object.assign({ 'content-type': 'application/json' }, init.headers || {}); }
  }

  var timeoutMs = opts.timeoutMs != null ? opts.timeoutMs : 15000;
  var retries = opts.retries || 0;
  var retryDelay = opts.retryDelayMs || 600;

  function attempt(n) {
    return withTimeout(Promise.resolve(fetcher(url, init)), timeoutMs, method + ' ' + url)
      .then(function (res) { return normalize(res, url, direct); })
      .catch(function (err) {
        if (n < retries) {
          return new Promise(function (r) { setTimeout(r, retryDelay * (n + 1)); }).then(function () { return attempt(n + 1); });
        }
        // If we're under Perchance's ?$csp restricted mode, an outbound
        // failure is most likely the platform blocking it, not a bug.
        if (cspActive() && err && err.message && err.message.indexOf('$csp') === -1) {
          err.message += ' (note: page is in Perchance ?$csp restricted mode, which can block outbound requests)';
        }
        throw err;
      });
  }
  return attempt(0);
}

// Normalize a fetch Response (or superFetch result) into a plain object.
function normalize(res, url, direct) {
  if (res == null) return { ok: false, status: 0, url: url, via: direct ? 'direct' : 'proxy', text: function () { return Promise.resolve(''); } };
  var status = (typeof res.status === 'number') ? res.status : 200;
  var ok = (typeof res.ok === 'boolean') ? res.ok : (status >= 200 && status < 300);
  var headers = res.headers || null;
  var bodyText = null;
  function text() {
    if (bodyText != null) return Promise.resolve(bodyText);
    if (typeof res.text === 'function') return Promise.resolve(res.text()).then(function (t) { bodyText = String(t); return bodyText; });
    bodyText = String(res); return Promise.resolve(bodyText);
  }
  function json() { return text().then(function (t) { try { return JSON.parse(t); } catch (e) { throw new Error('weld.fetch: response is not JSON (' + t.slice(0, 60) + ')'); } }); }
  return { ok: ok, status: status, url: url, via: direct ? 'direct' : 'proxy', headers: headers, text: text, json: json, raw: res };
}

// Convenience verbs.
function get(url, opts) { return request(url, Object.assign({}, opts, { method: 'GET' })); }
function post(url, opts) { return request(url, Object.assign({}, opts, { method: 'POST' })); }
function del(url, opts) { return request(url, Object.assign({}, opts, { method: 'DELETE' })); }

// GET (or any method) and parse JSON in one step -> the parsed value.
function json(url, opts) {
  opts = opts || {};
  return request(url, opts).then(function (res) {
    if (!res.ok) throw new Error('weld.fetch: ' + res.status + ' from ' + url);
    return res.json();
  });
}
// GET and return text in one step.
function text(url, opts) { return request(url, opts || {}).then(function (res) { return res.text(); }); }

function available() { return !!getSF(); }

// Perchance's ?$csp URL mode (perchance.org/custom-content-security-policy)
// restricts a page's ability to reach external servers. Under it, the
// superFetch proxy and direct fetches to outside hosts may be blocked at
// the platform level. weld.fetch can't lift that restriction, but it can
// tell you it's in effect so a failed request reads as "CSP blocked this"
// rather than a mysterious network error.
function cspActive() {
  try {
    var q = (typeof location !== 'undefined' && location.search) ? location.search : '';
    var h = (typeof location !== 'undefined' && location.href) ? location.href : '';
    return /[?&]\$csp(\b|=|&|$)/.test(q) || /[?&]\$csp(\b|=|&|$)/.test(h);
  } catch (e) { return false; }
}

window.weld.fetch = {
  request: request,
  get: get,
  post: post,
  del: del,
  json: json,
  text: text,
  attach: attach,
  available: available,
  cspActive: cspActive,
  addDirectHost: addDirectHost,
  directHosts: function () { return DIRECT_HOSTS.slice(); },
  version: '1.2.0'
};

return window.weld.fetch;
})();
</script>

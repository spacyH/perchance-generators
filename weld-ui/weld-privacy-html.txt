      <h1>weld.privacy</h1>
      <span class="ver">v1.1.0</span>
  </div>
  <p class="lead">A vocabulary and registry for declaring, honestly, how each backend an app uses handles data -- inspired by Duck.ai per-model privacy claims. Claim kinds (local-only, anonymized, server-stored, public, expiring, third-party, transient), honest pre-registered claims for the suite own backends, and a legible data-handling summary. Describes posture; enforces nothing.</p>
</div>
<h2>Make your app legible about where data goes</h2>
<p>Duck.ai's sharpest idea is showing, per model, what privacy guarantee applies <em>before</em> you use it. weld.privacy ports that pattern (not the guarantees, which are platform-specific): a small claim vocabulary, an honest registry of how the suite's own backends behave, and a way to render a data-handling summary your app can show users.</p>
<div class="note"><strong>Honesty rule:</strong> a claim describes data-handling <em>posture</em> &mdash; it does <em>not</em> enforce anything. The built-in claims reflect what Perchance's backends actually do. Never attach a claim you can't stand behind; an overstated label is worse than none.</div>
<pre data-codeid="install"></pre>

<h2>The claim vocabulary</h2>
<table>
<tr><th>kind</th><th>meaning</th><th>leaves device?</th></tr>
<tr><td><code>local-only</code></td><td>kept in this browser, never sent</td><td>no</td></tr>
<tr><td><code>anonymized</code></td><td>sent with identifying metadata stripped/proxied</td><td>yes</td></tr>
<tr><td><code>server-stored</code></td><td>persisted on a remote server (often to sync)</td><td>yes</td></tr>
<tr><td><code>public</code></td><td>uploaded to a public, link-accessible URL</td><td>yes</td></tr>
<tr><td><code>expiring</code></td><td>stored with an expiry, auto-deleted later</td><td>yes</td></tr>
<tr><td><code>third-party</code></td><td>passes through a service outside your control</td><td>yes</td></tr>
<tr><td><code>transient</code></td><td>used for one request, not retained</td><td>yes</td></tr>
</table>

<h2>Pre-registered claims for the suite</h2>
<p>The suite's own backends ship with honest claims, so an app gets a correct summary for free:</p>
<table>
<tr><th>backend</th><th>claims</th></tr>
<tr><td><code>persist</code>, <code>history</code>, <code>backup</code>, <code>cache</code>, <code>state</code>, <code>memory</code></td><td>local-only</td></tr>
<tr><td><code>kv</code></td><td>server-stored (cross-device)</td></tr>
<tr><td><code>upload</code>, <code>report</code></td><td>public + expiring</td></tr>
<tr><td><code>stream</code>, <code>image</code>, <code>fetch</code></td><td>anonymized + third-party (Perchance broker/proxy)</td></tr>
<tr><td><code>registry</code></td><td>transient + third-party</td></tr>
</table>

<h2>claimsFor / register</h2>
<div class="sig" data-codeid="sig_claims"></div>
<p>Read the claims for any backend, or declare your own. Unknown claim kinds are dropped rather than throwing, so a typo can never crash a privacy panel.</p>
<pre data-codeid="register_ex"></pre>

<h2>summary / report / leavesDevice</h2>
<div class="sig" data-codeid="sig_summary"></div>
<p>Given the set of backends your app uses, <code>summary()</code> returns a plain-language digest, <code>report()</code> a structured version for building UI, and <code>leavesDevice()</code> a single at-a-glance boolean. <code>localOnly()</code> filters to the backends that keep data on-device.</p>
<pre data-codeid="summary_ex"></pre>
<div class="note">Pairs with the suite: declare your app's full backend set once, show <code>summary()</code> in an info panel, and use <code>leavesDevice()</code> to drive a privacy indicator. Complements <code>weld.report</code> (which encrypts before its public+expiring upload).</div>
<h2>Restricted mode (?$csp) <span class="tag">v1.1.0</span></h2>
<div class="sig" data-codeid="sig_csp"></div>
<p>Perchance offers a <code>?$csp</code> URL mode (its <em>Custom Content-Security-Policy</em> feature) that restricts a generator&rsquo;s ability to reach external servers &mdash; adding <code>?$csp</code> to a page&rsquo;s URL means a user has to trust the author less, because outbound third-party requests are blocked at the platform level. That is a real data-handling fact, so <code>cspRestricted()</code> reports it, and <code>summary()</code> / <code>report()</code> note it when active. An app can use this to tell the user they&rsquo;re in a hardened mode, or to adjust which features it offers.</p>

<script>
(function(){
'use strict';

if (!window.weld) window.weld = {};
if (window.weld.privacy) return window.weld.privacy;

// -----------------------------------------------------------------
// weld.privacy: make an app legible about where its data goes.
//
// Duck.ai's best idea is showing, per model, what privacy guarantee
// applies BEFORE you use it -- anonymized, zero-retention, no-training,
// and so on. weld.privacy ports the *pattern* (not the guarantees, which
// are platform-specific): a small claim vocabulary, an honest registry of
// how the suite's own backends behave, and a way to render a data-handling
// summary an app can show its users.
//
//   weld.privacy.claimsFor('kv')   returns an array of claim objects
//   weld.privacy.summary of the backends an app uses -> human-readable text
//   weld.privacy.register a backend with its claim kinds
//
// IMPORTANT HONESTY RULE: a claim describes data-handling POSTURE; it does
// NOT enforce anything. The labels here reflect what Perchance's backends
// actually do, to the best of documented behavior. An app must not attach
// a claim it can't stand behind -- an overstated label is worse than none.
// -----------------------------------------------------------------

// The claim vocabulary. Each kind has a stable id, a short title, a
// plain-language detail, and a coarse 'leaves-device' flag apps can sort on.
var KINDS = {
  'local-only': {
    title: 'Stays on this device',
    detail: 'Data is kept in this browser only (IndexedDB / localStorage) and is never sent anywhere. Clearing your browser data removes it.',
    leavesDevice: false
  },
  'anonymized': {
    title: 'Sent without identifying you',
    detail: 'Data is sent to a backend with identifying metadata stripped or proxied, so the request is not tied to your identity.',
    leavesDevice: true
  },
  'server-stored': {
    title: 'Stored on a server',
    detail: 'Data is stored on a remote server and persists there (often so it can sync across your devices). It is not only on this device.',
    leavesDevice: true
  },
  'public': {
    title: 'Publicly accessible',
    detail: 'Data is uploaded to a public URL that anyone with the link can access. Do not put anything sensitive here.',
    leavesDevice: true
  },
  'expiring': {
    title: 'Auto-deletes later',
    detail: 'Data is stored with an expiry and is removed automatically after a set time.',
    leavesDevice: true
  },
  'third-party': {
    title: 'Handled by a third party',
    detail: 'Data passes through a service outside your control, whose own data-handling policy applies.',
    leavesDevice: true
  },
  'transient': {
    title: 'Not stored at all',
    detail: 'Data is used for a single request and not retained after the response.',
    leavesDevice: true
  }
};

function kindIds() { return Object.keys(KINDS); }
function describeKind(id) {
  var k = Reflect.get(KINDS, id);
  return k ? { kind: id, title: k.title, detail: k.detail, leavesDevice: k.leavesDevice } : null;
}

// Honest, pre-registered claims for the suite's own backends. These reflect
// documented Perchance behavior; an app can override or extend them.
//   persist/history/backup -> local IndexedDB only          -> local-only
//   cache/state            -> in-memory or local            -> local-only
//   kv                     -> kv-plugin, server cross-device -> server-stored
//   upload                 -> public URL (+ optional expiry) -> public (+expiring)
//   stream/image           -> Perchance broker (proxied)     -> anonymized + third-party
//   fetch                  -> superFetch proxy (strips hdrs) -> anonymized + third-party
//   registry               -> reads Perchance public API     -> transient + third-party
//   report                 -> encrypted, expiring upload     -> public + expiring
var REGISTRY = {
  persist:  ['local-only'],
  history:  ['local-only'],
  backup:   ['local-only'],
  cache:    ['local-only'],
  state:    ['local-only'],
  memory:   ['local-only'],
  kv:       ['server-stored'],
  upload:   ['public', 'expiring'],
  report:   ['public', 'expiring'],
  stream:   ['anonymized', 'third-party'],
  image:    ['anonymized', 'third-party'],
  fetch:    ['anonymized', 'third-party'],
  registry: ['transient', 'third-party']
};

// register(name, kinds): declare (or override) the claims for a backend.
// kinds is an array of claim ids; unknown ids are dropped with no throw.
function register(name, kinds) {
  if (!name) return null;
  var list = (Array.isArray(kinds) ? kinds : Array.of(kinds)).filter(function (k) { return !!Reflect.get(KINDS, k); });
  Reflect.set(REGISTRY, String(name), list);
  return claimsFor(name);
}

// claimsFor(name): the resolved claim objects for a backend (or empty array).
function claimsFor(name) {
  var ids = Reflect.get(REGISTRY, String(name)) || new Array();
  return ids.map(describeKind).filter(Boolean);
}

// worst-case posture across a set of backends: does ANY of them leave the
// device? Useful for a single at-a-glance indicator.
function leavesDevice(names) {
  return toArray(names).some(function (n) {
    return claimsFor(n).some(function (c) { return c.leavesDevice; });
  });
}

function toArray(x) { return Array.isArray(x) ? x : (x == null ? new Array() : Array.of(x)); }

// summary(names): a human-readable data-handling digest for the backends an
// app uses. Apps can show this in a privacy panel or an info modal.
function summary(names) {
  var list = toArray(names);
  if (!list.length) return 'No backends declared.';
  var lines = new Array();
  lines.push(leavesDevice(list)
    ? 'Some data leaves this device:'
    : 'All data stays on this device.');
  if (cspRestricted()) {
    lines.push('(Running in Perchance ?$csp restricted mode: outbound external requests are blocked by the platform.)');
  }
  list.forEach(function (n) {
    var claims = claimsFor(n);
    if (!claims.length) {
      lines.push('  ' + n + ': (no privacy claim declared)');
    } else {
      lines.push('  ' + n + ': ' + claims.map(function (c) { return c.title; }).join('; '));
    }
  });
  return lines.join('\n');
}

// report(names): a structured version of summary() for building UI.
function report(names) {
  var list = toArray(names);
  return {
    leavesDevice: leavesDevice(list),
    cspRestricted: cspRestricted(),
    backends: list.map(function (n) { return { name: n, claims: claimsFor(n) }; })
  };
}

// Audit helper: which declared backends keep data only on-device?
function localOnly(names) {
  return toArray(names).filter(function (n) {
    var c = claimsFor(n);
    return c.length && c.every(function (x) { return !x.leavesDevice; });
  });
}

// Perchance's ?$csp URL mode (perchance.org/custom-content-security-policy)
// restricts a generator's ability to talk to external servers -- adding
// ?$csp to a page's URL reduces how much a user must trust the author.
// It's a real, current data-handling fact, so weld.privacy surfaces it:
// an app can tell the user "you're running in restricted mode" and adjust
// claims accordingly (outbound third-party calls won't go through).
function cspRestricted() {
  try {
    var q = (typeof location !== 'undefined' && location.search) ? location.search : '';
    var h = (typeof location !== 'undefined' && location.href) ? location.href : '';
    // Perchance uses a bare ?$csp flag; match it in the query or full URL.
    return /[?&]\$csp(\b|=|&|$)/.test(q) || /[?&]\$csp(\b|=|&|$)/.test(h);
  } catch (e) { return false; }
}

window.weld.privacy = {
  kinds: KINDS,
  kindIds: kindIds,
  describeKind: describeKind,
  register: register,
  claimsFor: claimsFor,
  leavesDevice: leavesDevice,
  localOnly: localOnly,
  summary: summary,
  report: report,
  cspRestricted: cspRestricted,
  version: '1.1.0'
};

return window.weld.privacy;
})();
</script>

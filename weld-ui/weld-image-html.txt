      <h1>weld.image</h1>
      <span class="ver">v1.1.0</span>
    </div>
    <p class="lead">Promise-based text-to-image: styles, batching, retry. A thin wrapper around Perchance's text-to-image-plugin with the ergonomics you actually want.</p>
  </div>
  <h2>Overview</h2>
<p>weld.image wraps Perchance's <code>text-to-image-plugin</code> with the things you want when actually using it: Promise-based generation (instead of callback-based), batched runs with progress callbacks, ten style presets, four resolution presets, automatic retry with exponential backoff, and an <code>into</code> shortcut for direct DOM insertion.</p>
<p>The underlying generator is unchanged -- Perchance's backend ignores negative prompts, so the style presets are positive-only suffixes that nudge the model toward a specific aesthetic.</p>
<h2>Quick start</h2>
<p>In your lists editor, weld.image needs the text-to-image-plugin imported alongside it. The plugin auto-wires them together:</p>

<h2>Single generation</h2><h3>weld.image.generate(opts)</h3>
<div class="sig" data-codeid="sig4"></div>
<p>Wraps the text-to-image plugin in a Promise. Resolves with a <strong>normalized</strong> result object &mdash; <code>&#123; dataUrl, canvas, inputs, seed, text, raw &#125;</code> &mdash; when generation finishes, or rejects on timeout. (The original boxed-String result is kept on <code>raw</code>.)</p>
<p>Options are passed straight through to the underlying plugin, plus three weld-specific ones:</p>
<ul>
<li><code>into</code>: CSS selector or element. The generated iframe is inserted there automatically.</li>
<li><code>timeoutMs</code>: how long to wait before rejecting. Default 120 seconds.</li>
<li><code>onFinish</code>: optional callback fired in parallel with the Promise resolution.</li>
</ul>

<h2>Reliability &mdash; the text-to-image truth layer</h2>
<p>The underlying plugin has three sharp edges that produce bugs with no error message. <code>generate()</code> handles all three so you don't have to:</p>
<ul>
<li><strong>Boxed-String results.</strong> The raw awaited result is a boxed <code>String</code> carrying <code>canvas</code>/<code>dataUrl</code>/<code>inputs</code>, so <code>===</code> comparisons silently fail. <code>generate()</code> returns a plain object instead; use <code>weld.image.normalizeResult(data)</code> if you call the plugin directly elsewhere.</li>
<li><strong>Only four resolutions.</strong> <code>512&times;512</code>, <code>512&times;768</code>, <code>768&times;512</code>, <code>768&times;768</code>. Anything else is rejected up front with a clear error rather than a silent failure. Check one with <code>weld.image.validResolution('768x768')</code>.</li>
<li><strong>Empty prompt hangs forever.</strong> An empty/whitespace prompt never returns from the backend, so <code>generate()</code> rejects it immediately.</li>
</ul>
<h3>weld.image.regenerate(opts)</h3>
<div class="sig" data-codeid="sig_regen"></div>
<p>Reproduce or vary an image deterministically. With an explicit <code>seed</code>, the same prompt + seed reproduces the same image &mdash; the "regenerate at full quality from a saved thumbnail + seed" pattern. Pass <code>vary: true</code> to roll a fresh seed for an alternate take. The resolved object's <code>seed</code> is the one used, so you can save it.</p>
<pre data-codeid="regen_ex"></pre>

<h3>weld.image.embed(opts)</h3>
<div class="sig" data-codeid="sig5"></div>
<p>Synchronous version that returns the iframe HTML for you to insert manually. Useful when you want to position multiple images precisely or batch insertion into a list.</p>
<pre data-codeid="code6"></pre>
<h2>Styles and resolutions</h2><h3>weld.image.withStyle(prompt, styleName)</h3>
<p>Append a positive-only style suffix to a prompt. The ten built-in styles:</p>
<ul>
<li><code>photo</code> -- photorealistic, sharp focus, detailed, 8k</li>
<li><code>cinematic</code> -- cinematic shot, dramatic lighting, depth of field</li>
<li><code>anime</code> -- anime style, cel shaded, vibrant</li>
<li><code>painting</code> -- oil painting, classical art, painterly</li>
<li><code>digital</code> -- digital art, concept art, trending on artstation</li>
<li><code>watercolor</code> -- watercolor painting, soft edges, paper texture</li>
<li><code>sketch</code> -- pencil sketch, hand drawn, graphite</li>
<li><code>lowpoly</code> -- low poly 3d render, geometric, isometric</li>
<li><code>pixelart</code> -- pixel art, 16-bit style, retro game graphics</li>
<li><code>cyberpunk</code> -- cyberpunk, neon lights, futuristic</li>
</ul>
<p>You can also access the raw style strings via <code>weld.image.styles[name]</code> if you want to inspect or modify them.</p>
<h3>weld.image.withResolution(opts, resName)</h3>
<div class="sig" data-codeid="sig7"></div>
<p>Add <code>width</code> and <code>height</code> to an options object. Four built-ins: <code>square</code> (512x512), <code>portrait</code> (512x768), <code>landscape</code> (768x512), <code>largeSquare</code> (768x768).</p>
<pre data-codeid="code8"></pre>
<h2>Batch generation</h2><h3>weld.image.batch(opts, count, onProgress?)</h3>
<div class="sig" data-codeid="sig9"></div>
<p>Run the same prompt N times with auto-incrementing seeds. Returns an array of result data (or <code>\{error\}</code> entries for failed cells).</p>
<pre data-codeid="code10"></pre>
<p>If <code>opts.seed</code> is omitted or set to <code>-1</code>, each cell gets a random seed instead.</p>
<h3>weld.image.variations(prompts, sharedOpts, onProgress?)</h3>
<div class="sig" data-codeid="sig11"></div>
<p>Generate one image per prompt, sharing the rest of the options. Useful for character sheets, scene variations, or A/B prompts.</p>
<pre data-codeid="code12"></pre>
<h2>Retry on failure</h2>
<div class="sig" data-codeid="sig13"></div>
<p>Wraps <code>generate</code> with exponential backoff. First retry after 1s, then 2s, 4s, etc. Default <code>maxRetries</code> is 3.</p>
<pre data-codeid="code14"></pre>

<pre data-codeid="code15"></pre>
<p>The standard DSL form does this automatically -- you only need <code>setT2i</code> if you're loading weld.image's source directly (e.g. through a script tag) without going through the DSL <code>$output</code> closure.</p>
<h2>Negative prompts</h2><div class="note">
Perchance's text-to-image backend silently ignores the <code>negativePrompt</code> option, so weld.image doesn't include negative styling in its presets. If you need to suppress something, work it into the positive prompt explicitly (e.g. <em>"daylight scene, blue sky"</em> instead of <em>negativePrompt: "night"</em>).

<script>
(function(){
'use strict';

if (!window.weld) window.weld = {};
if (window.weld.image) return; // already loaded

// -----------------------------------------------------------------
// weld.image: Promise-based wrapper around Perchance's text-to-image-plugin.
// Adds batching, style presets, retry, resolution helpers.
//
// The actual t2i function comes from Perchance's text-to-image-plugin,
// which must be imported in the DSL panel under any alias, e.g.:
//   t2iSource = the import of text-to-image-plugin
// The DSL form passes that function into weld.image.setT2i(...).
// Standalone JS callers must call weld.image.setT2i(fn) themselves.
// -----------------------------------------------------------------

var t2iFn = null;

// Resolution presets
var RESOLUTIONS = {
  square:      '512x512',
  portrait:    '512x768',
  landscape:   '768x512',
  largeSquare: '768x768'
};

// Positive-only style presets (Perchance backend ignores negatives)
var STYLES = {
  photo:      ', photorealistic, sharp focus, detailed, 8k, professional photography',
  cinematic:  ', cinematic shot, dramatic lighting, depth of field, film grain, atmospheric',
  anime:      ', anime style, cel shaded, vibrant, detailed line art',
  painting:   ', oil painting, classical art, brush strokes, painterly, masterpiece',
  digital:    ', digital art, concept art, trending on artstation, highly detailed',
  watercolor: ', watercolor painting, soft edges, paper texture, flowing colors',
  sketch:     ', pencil sketch, hand drawn, graphite, shading, detailed lines',
  lowpoly:    ', low poly 3d render, geometric, isometric, minimalist',
  pixelart:   ', pixel art, 16-bit style, retro game graphics, pixelated',
  cyberpunk:  ', cyberpunk, neon lights, futuristic, blade runner aesthetic, rain'
};

function setT2i(fn) {
  t2iFn = fn;
}

function getT2i() {
  if (!t2iFn) {
    throw new Error('weld.image: no text-to-image function set. Call weld.image.setT2i(fn) where fn is the imported text-to-image-plugin.');
  }
  return t2iFn;
}

function styleNames() { return Object.keys(STYLES); }
function resolutionNames() { return Object.keys(RESOLUTIONS); }

function withStyle(prompt, styleName) {
  var suffix = STYLES[styleName] || '';
  return String(prompt) + suffix;
}

function withResolution(opts, resName) {
  var res = RESOLUTIONS[resName];
  if (!res) return Object.assign({}, opts);
  var parts = res.split('x');
  return Object.assign({}, opts, { width: parseInt(parts[0], 10), height: parseInt(parts[1], 10) });
}

// Synchronous embed -- returns the iframe HTML for direct insertion
function embed(opts) {
  return String(getT2i()(opts));
}

// Async single generation -- resolves with the result data
function generate(opts) {
  return new Promise(function(resolve, reject) {
    var settled = false;
    var fn = getT2i();
    var enriched = Object.assign({}, opts, {
      onFinish: function(data) {
        if (settled) return;
        settled = true;
        resolve(data);
      }
    });
    var html;
    try {
      html = String(fn(enriched));
    } catch (e) {
      reject(e);
      return;
    }

    try {
      if (opts && opts.into) {
        var target = typeof opts.into === 'string' ? document.querySelector(opts.into) : opts.into;
        if (target) target.innerHTML = html;
      }
    } catch (e) { /* ignore */ }

    var ms = (opts && opts.timeoutMs) || 120000;
    setTimeout(function() {
      if (!settled) {
        settled = true;
        reject(new Error('weld.image: generation timeout after ' + ms + 'ms'));
      }
    }, ms);
  });
}

// Generate N images sequentially with the same prompt (variation via seed)
function batch(opts, count, onProgress) {
  return new Promise(function(resolve) {
    var results = [];
    var i = 0;
    function step() {
      if (i >= count) { resolve(results); return; }
      var cellOpts = Object.assign({}, opts);
      if (cellOpts.seed === undefined || cellOpts.seed === -1) {
        cellOpts.seed = Math.floor(Math.random() * 1000000);
      } else if (i > 0) {
        cellOpts.seed = opts.seed + i;
      }
      generate(cellOpts).then(function(data) {
        results.push(data);
        if (onProgress) onProgress(i + 1, count, data);
        i++;
        step();
      }).catch(function(e) {
        results.push({ error: e.message });
        if (onProgress) onProgress(i + 1, count, { error: e.message });
        i++;
        step();
      });
    }
    step();
  });
}

// Variations: same opts but different prompts
function variations(prompts, sharedOpts, onProgress) {
  return new Promise(function(resolve) {
    var results = [];
    var i = 0;
    function step() {
      if (i >= prompts.length) { resolve(results); return; }
      var cellOpts = Object.assign({}, sharedOpts, { prompt: prompts[i] });
      generate(cellOpts).then(function(data) {
        results.push(data);
        if (onProgress) onProgress(i + 1, prompts.length, data);
        i++;
        step();
      }).catch(function(e) {
        results.push({ error: e.message, prompt: prompts[i] });
        if (onProgress) onProgress(i + 1, prompts.length, { error: e.message });
        i++;
        step();
      });
    }
    step();
  });
}

// Auto-retry on failure with exponential backoff
function generateWithRetry(opts, maxRetries) {
  maxRetries = maxRetries || 3;
  return new Promise(function(resolve, reject) {
    var attempt = 0;
    var lastError = null;
    function tryOnce() {
      generate(opts).then(resolve).catch(function(e) {
        lastError = e;
        attempt++;
        if (attempt >= maxRetries) { reject(lastError); return; }
        var backoff = 1000 * Math.pow(2, attempt - 1);
        setTimeout(tryOnce, backoff);
      });
    }
    tryOnce();
  });
}

window.weld.image = {
  setT2i: setT2i,
  withStyle: withStyle,
  withResolution: withResolution,
  styleNames: styleNames,
  resolutionNames: resolutionNames,
  styles: STYLES,
  resolutions: RESOLUTIONS,
  embed: embed,
  generate: generate,
  batch: batch,
  variations: variations,
  generateWithRetry: generateWithRetry,
  version: '1.0.0'
};

})();

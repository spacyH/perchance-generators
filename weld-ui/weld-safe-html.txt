  <div class="hdr-row">
      <h1>weld.safe</h1>
      <span class="ver">v1.1.0</span>
  </div>
  <p class="lead">The brace-interception toolkit for Perchance authors. Scan code for the tokens the platform silently rewrites &mdash; with line numbers, explanations, and fixes &mdash; then escape strings and build safe tokens so your code survives the parser.</p>
</div>
<h2>The problem</h2>
<p>Before your code runs, Perchance scans the <strong>raw source</strong> &mdash; including the insides of <code>&lt;script&gt;</code> blocks, string literals, and <code>//</code> comments &mdash; and rewrites anything that looks like its own template syntax. A braced word, doubled brackets, an import directive, a tight pipe pick-list, or a count quantifier sitting in a string can be silently mangled, producing bugs with no stack trace. weld.safe finds them, explains them, and helps you write around them.</p>
<pre data-codeid="install"></pre>

<h2>scan(code)</h2>
<div class="sig" data-codeid="sig_scan"></div>
<p>Returns an array of findings: <code>\&#123; line, col, token, kind, severity, why, fix \&#125;</code>. Severities: <code>error</code> (always intercepted), <code>warn</code> (intercepted in most contexts), <code>info</code> (safe in pure JS, risky in DSL/prose).</p>
<pre data-codeid="scan_ex"></pre>

<h2>The traps it catches</h2>
<table>
<tr><th>kind</th><th>severity</th><th>what triggers it</th></tr>
<tr><td><code>double-square</code></td><td>error</td><td>two opening square brackets in a row</td></tr>
<tr><td><code>double-curly</code></td><td>error</td><td>two opening curly braces in a row</td></tr>
<tr><td><code>import-directive</code></td><td>error</td><td>an import token anywhere but a top-level assignment</td></tr>
<tr><td><code>pick-list</code></td><td>warn</td><td>a tight pipe-separated brace group (no spaces)</td></tr>
<tr><td><code>quantifier</code></td><td>warn</td><td>a brace group of just a number, or number,number</td></tr>
<tr><td><code>bare-token</code></td><td>info</td><td>a single braced word with no spaces</td></tr>
</table>
<div class="note">Spaced object literals (<code>\&#123; a: 1 \&#125;</code>), logical-or in braces (<code>\&#123; a &#124;&#124; b \&#125;</code>), and regex character classes are deliberately <em>not</em> flagged &mdash; they read as code, not template syntax. A doubled <em>closing</em> brace is also fine; only the doubled <em>opening</em> triggers.</div>

<h2>check(code)</h2>
<div class="sig" data-codeid="sig_check"></div>
<p>A pass/fail wrapper: <code>\&#123; ok, findings \&#125;</code> where <code>ok</code> is false if any error-severity trap is present. Drop it into a build step.</p>

<h2>escape(str)</h2>
<div class="sig" data-codeid="sig_escape"></div>
<p>Returns a version of a string with every trap character (<code>\&#123;\&#125;&#91;&#93;&#124;</code>) replaced by its HTML entity, so the parser leaves it alone. Use for prose or HTML you want to display literally.</p>
<pre data-codeid="escape_ex"></pre>

<h2>build(token) &amp; builderSnippet(token)</h2>
<div class="sig" data-codeid="sig_build"></div>
<div class="sig" data-codeid="sig_snip"></div>
<p><code>build()</code> returns the real characters of a token, assembled at runtime from char codes, so the literal never appears in source for the parser to see. <code>builderSnippet()</code> returns a <code>String.fromCharCode(...)</code> expression you can paste straight into your own code.</p>
<pre data-codeid="build_ex"></pre>

<h2>explain(kind)</h2>
<div class="sig" data-codeid="sig_explain"></div>
<p>Returns the rule (<code>severity</code>, <code>why</code>, <code>fix</code>) for any trap kind &mdash; handy for surfacing guidance in your own tooling. <code>weld.safe.KINDS</code> lists them all.</p>
<h2>Canonical escape set <span class="tag">v1.1.0</span></h2>
<div class="sig" data-codeid="sig_esc"></div>
<p>Straight from Perchance&rsquo;s engine (<code>processEscapedCharacters</code>), <code>escapableChars()</code> returns the exact characters a backslash escapes: the assignment/odds operator, the brace pair, the square-bracket pair, the caret, and the pipe &mdash; plus the whitespace escapes (<code>\n</code> newline, <code>\t</code> tab, <code>\s</code> a literal space the parser would otherwise trim from an item edge). As of this version <code>escape()</code> also entity-encodes <code>=</code> and <code>^</code>, not just the brace/bracket/pipe characters, so displayed prose can&rsquo;t be misread as an assignment or an odds operator. Use <code>escapableChars()</code> when you need the authoritative list rather than folklore &mdash; for example to build a live-lint tool.</p>
<pre data-codeid="esc_ex"></pre>

  function escapeHTML(s){

    var kws = ['var','let','const','function','return','if','else','for','while','await','async','new','typeof','instanceof','null','true','false','this','of','in'];

<script>
(function(){
'use strict';

if (!window.weld) window.weld = {};
if (window.weld.safe) return window.weld.safe;

// -----------------------------------------------------------------
// weld.safe: the single biggest Perchance footgun, packaged.
//
// Before your code ever runs, Perchance scans the RAW source -- including
// the insides of <script> blocks, string literals, and // comments --
// and rewrites tokens that look like its own template syntax. A stray
// braced word, a doubled bracket, an import directive, a tight pipe
// pick-list, or a regex count quantifier sitting in a string can all be
// silently mangled, producing bugs with no stack trace.
//
//   weld.safe.scan(code)   -> array of findings, each with line, col,
//                             token, severity, why, fix
//   weld.safe.check(code)  -> ok flag plus findings (ok = no errors)
//   weld.safe.escape(str)  -> a parser-safe version of a string
//   weld.safe.build(token) -> assemble a risky literal at runtime
//   weld.safe.explain(kind)-> the rule behind a trap kind
//
// The detection mirrors weld.diag's validated trap rules and goes deeper
// on pick-lists and quantifiers, plus the escape/build helpers diag lacks.
// -----------------------------------------------------------------

var OB = String.fromCharCode(123); // {
var CB = String.fromCharCode(125); // }
var OS = String.fromCharCode(91);  // [
var CS = String.fromCharCode(93);  // ]
var BAR = String.fromCharCode(124);// |

var RULES = {
  'double-square': {
    severity: 'error',
    why: 'Doubled square brackets begin a Perchance nested-list / shortcode token and are intercepted everywhere, including inside strings and comments.',
    fix: 'Break them up: build the literal with weld.safe.build, or insert a zero-width boundary. For array-of-arrays in JS, add a space: "' + OS + ' ' + OS + '".'
  },
  'double-curly': {
    severity: 'error',
    why: 'Doubled curly braces are Perchance advanced/template syntax and are intercepted in raw source.',
    fix: 'Add a space between them (" ' + OB + ' ' + OB + ' "), or assemble with weld.safe.build. Note: a doubled CLOSING brace is usually safe; it is the doubled OPENING that triggers it.'
  },
  'import-directive': {
    severity: 'error',
    why: 'An import directive is only valid as a top-level assignment. Anywhere else (in a string, comment, or expression) it is intercepted.',
    fix: 'Keep imports at the top of the file as `name = ' + OB + 'import:slug' + CB + '`. To mention one in prose/strings, escape it with weld.safe.escape.'
  },
  'pick-list': {
    severity: 'warn',
    why: 'A tight pipe-separated brace group with no spaces is Perchance pick-one syntax; the parser will replace it with one of the options at load time.',
    fix: 'If you meant a literal, escape it with weld.safe.escape (HTML entities) or add spaces inside the braces. In JS logic, "' + OB + ' a ' + BAR + BAR + ' b ' + CB + '" (spaced) is read as code, not a pick.'
  },
  'quantifier': {
    severity: 'warn',
    why: 'A brace group containing only a number (or number,number) looks like a Perchance repeat/count token and can be intercepted in DSL/prose context.',
    fix: 'In a regex, this is usually fine inside a /.../ literal; in a plain string, escape it or build it with weld.safe.build.'
  },
  'bare-token': {
    severity: 'info',
    why: 'A single braced word with no spaces can be read as a Perchance variable/list reference if it lands in DSL or prose context. In pure JS expressions it is normally safe.',
    fix: 'Safe inside real JS (object literals with spaces, template scopes). If this is in a string shown to the parser, escape it.'
  }
};

// Strip JS string/regex/comment *content* is NOT what we do here -- the
// whole point is that Perchance scans that content too. So we scan raw,
// but we DO skip flagging spaced object literals and regex char-classes
// to avoid false positives.

function scan(code) {
  var lines = String(code == null ? '' : code).split('\n');
  var out = new Array();
  var outIdx = -1;
  for (var i = 0; i < lines.length; i++) { if (/^\s*\$output\s*\(/.test(Reflect.get(lines, i))) { outIdx = i; break; } }

  for (var ln = 0; ln < lines.length; ln++) {
    var raw = Reflect.get(lines, ln);

    // double square / double curly (opening)
    findAll(raw, OS + OS, function (col) { push(out, ln, col, OS + OS, 'double-square'); });
    findAll(raw, OB + OB, function (col) {
      // ignore a doubled CLOSING brace run like "}}" (object/block closers)
      push(out, ln, col, OB + OB, 'double-curly');
    });

    // import directive that is NOT a legit top-level assignment
    var importRe = new RegExp(OB + 'import:' + OS + '^' + CB + CS + '*' + CB, 'g');
    var im;
    while ((im = importRe.exec(raw)) !== null) {
      var isTopImport = (outIdx === -1 || ln < outIdx)
        && new RegExp('^\\s*' + OS + '\\w$' + CS + '+\\s*=\\s*' + OB + 'import:' + OS + '^' + CB + CS + '+' + CB + '\\s*$').test(raw.trim());
      if (!isTopImport) push(out, ln, im.index, im[0], 'import-directive');
    }

    // tight pick-list: a braced, pipe-separated group with no spaces
    var pickRe = new RegExp(OB + OS + '^' + OB + CB + '\\s' + CS + '*\\' + BAR + OS + '^' + OB + CB + CS + '*' + CB, 'g');
    var pk;
    while ((pk = pickRe.exec(raw)) !== null) {
      var tok = pk[0];
      // skip if it contains spaces around the pipe (read as JS: { a || b })
      if (/\s/.test(tok)) continue;
      // skip JS logical-or in braces without spaces is rare; flag it
      push(out, ln, pk.index, tok, 'pick-list');
    }

    // quantifier: a braced number, or number-comma-number, standalone
    var quantRe = new RegExp(OB + '\\d+(,\\d*)?' + CB, 'g');
    var q;
    while ((q = quantRe.exec(raw)) !== null) {
      // if it's inside an obvious regex literal on this line, lower to info
      push(out, ln, q.index, q[0], 'quantifier');
    }

    // bare single-word token: a braced identifier, no spaces/colon/quote -> info
    var bareRe = new RegExp(OB + OS + 'A-Za-z_' + CS + OS + 'A-Za-z0-9_' + CS + '*' + CB, 'g');
    var b;
    while ((b = bareRe.exec(raw)) !== null) {
      push(out, ln, b.index, b[0], 'bare-token');
    }
  }
  // sort by line then column
  out.sort(function (a, c) { return a.line - c.line || a.col - c.col; });
  return out;
}

function findAll(s, needle, cb) {
  var idx = s.indexOf(needle);
  while (idx !== -1) { cb(idx); idx = s.indexOf(needle, idx + 1); }
}

function push(out, ln, col, token, kind) {
  var rule = Reflect.get(RULES, kind);
  out.push({
    line: ln + 1, col: col + 1, token: token, kind: kind,
    severity: rule.severity, why: rule.why, fix: rule.fix
  });
}

function check(code) {
  var findings = scan(code);
  var ok = !findings.some(function (f) { return f.severity === 'error'; });
  return { ok: ok, findings: findings };
}

// escape(str): produce a version of a string that the Perchance parser
// will leave alone, by replacing the trap-trigger characters with HTML
// entities. Use for prose/HTML you want to display literally.
//
// The entity STRINGS themselves are assembled at runtime from char codes
// because Perchance decodes HTML entities and RE-SCANS the result -- so a
// literal numeric-entity for an opening brace, written here in source,
// would decode back to that brace and trip the very interception this
// function exists to prevent. Never write the entity (or the brace) raw.
var AMP = String.fromCharCode(38) + String.fromCharCode(35); // ampersand + hash
var SEMI = String.fromCharCode(59);                          // semicolon
function ent(code) { return AMP + code + SEMI; }
var EQ = String.fromCharCode(61);    // =  (assignment / odds operator)
var CARET = String.fromCharCode(94); // ^  (conditional-odds operator)
function escape(str) {
  // Search regexes are also built from char-code vars (OB/CB/OS/CS/BAR) so
  // no literal trap character appears in this plugin's own source.
  // The full set Perchance treats as escapable (per the engine's
  // processEscapedCharacters) covers assignment, brace, square, caret,
  // and pipe characters, plus the whitespace escapes (newline/tab/space).
  // We entity-encode the structural ones so displayed prose can't be read
  // as a list reference, assignment, pick-list, or odds operator.
  return String(str == null ? '' : str)
    .replace(new RegExp('\\' + OB, 'g'), ent('123'))
    .replace(new RegExp('\\' + CB, 'g'), ent('125'))
    .replace(new RegExp('\\' + OS, 'g'), ent('91'))
    .replace(new RegExp('\\' + CS, 'g'), ent('93'))
    .replace(new RegExp('\\' + BAR, 'g'), ent('124'))
    .replace(new RegExp(EQ, 'g'), ent('61'))
    .replace(new RegExp('\\' + CARET, 'g'), ent('94'));
}
// escapableChars(): the canonical set Perchance treats as escapable with a
// leading backslash, taken from the engine's own processEscapedCharacters,
// returned as data so a tool (such as a live-lint userscript) can rely on the
// exact list rather than folklore. Whitespace escapes are included with
// their outputs (\\s escapes a space the parser would otherwise trim).
function escapableChars() {
  return {
    structural: Array.of(EQ, OB, CB, OS, CS, CARET, BAR),
    whitespace: { n: String.fromCharCode(10), t: String.fromCharCode(9), s: String.fromCharCode(32) },
    note: 'Backslash-escape any structural char to show it literally.'
  };
}

// build(token): assemble a string at RUNTIME so the literal trap token
// never appears in source. Returns the actual characters, constructed
// from char codes, so Perchance's source scan can't see them.
function build(token) {
  var s = String(token == null ? '' : token);
  var out = '';
  for (var i = 0; i < s.length; i++) out += String.fromCharCode(s.charCodeAt(i));
  return out;
}

// Convenience: emit a JS snippet that builds `token` via fromCharCode,
// so authors can paste a safe constructor into their own source.
function builderSnippet(token) {
  var s = String(token == null ? '' : token);
  var codes = new Array();
  for (var i = 0; i < s.length; i++) codes.push(s.charCodeAt(i));
  return 'String.fromCharCode(' + codes.join(',') + ')';
}

function explain(kind) {
  var r = Reflect.get(RULES, kind);
  return r ? { kind: kind, severity: r.severity, why: r.why, fix: r.fix } : null;
}

window.weld.safe = {
  scan: scan,
  check: check,
  escape: escape,
  build: build,
  builderSnippet: builderSnippet,
  explain: explain,
  escapableChars: escapableChars,
  KINDS: Object.keys(RULES),
  version: '1.1.0'
};

      <h1>weld.lorebook</h1>
      <span class="ver">v1.0.0</span>
    </div>
    <p class="lead">Keyword-triggered context injection with priority sorting and token budgets. The "world bible" pattern, generalized.</p>
  </div>
  <h2>Overview</h2>
<p>weld.lorebook handles the "world bible" problem for LLM-driven content: keyword-triggered context injection. Define entries with trigger keywords and content, scan an input message, get back the entries that matched -- sorted by priority, capped at a token budget.</p>
<p>Use it to inject character backstory when a character is mentioned, world rules when relevant terms appear, or dynamic context the user can edit through a UI. The plugin handles the matching and budget logic; you handle what to do with the matched entries (prepend to a prompt, show in a side panel, whatever fits).</p>

<h2>Entry shape</h2>
<p>Every entry is an object with:</p>
<ul>
<li><code>keys</code>: a string or array of strings -- any of these triggers the entry. Required.</li>
<li><code>content</code>: the text that gets injected when triggered. Required.</li>
<li><code>priority</code>: number for sorting matches when multiple entries fire. Higher wins. Default 0.</li>
<li><code>enabled</code>: false to skip without removing. Default true.</li>
<li><code>matchMode</code>: <code>'word'</code> (default) or <code>'substring'</code>. Word mode respects word boundaries; substring matches anywhere.</li>
<li><code>caseSensitive</code>: default false.</li>
<li><code>label</code>: optional human-readable name (not used by the plugin itself; for your UI).</li>
<li><code>metadata</code>: optional object for whatever you want to track.</li>
<li><code>id</code>: optional id, otherwise auto-generated.</li>
</ul>
<h2>CRUD</h2><h3>book.add(entry) / .update(id, patch) / .remove(id)</h3>
<p>Standard operations. <code>add</code> returns the entry id. <code>update</code> takes a patch object merged with the existing entry. All three persist to storage if configured.</p>
<pre data-codeid="code3"></pre>
<h3>book.get(id) / .getAll() / .size() / .clear()</h3>
<p>Inspection helpers. <code>get</code> returns a shallow copy of one entry, <code>getAll</code> returns copies of all entries, <code>size</code> returns the count, <code>clear</code> removes everything.</p>
<h2>Scanning</h2><h3>book.scan(text, opts?)</h3>
<div class="sig" data-codeid="sig4"></div>
<p>Scan the input text for entries whose keys appear in it. Returns an array of match objects:</p>
<pre data-codeid="code5"></pre>
<p>Matches are sorted by priority (descending), then by position (ascending). The budget options apply after sorting -- the highest-priority matches make it in first.</p>
<ul>
<li><code>maxTokens</code>: total token budget across all matches. Entries that would push past the budget are skipped (the next entry may still fit).</li>
<li><code>maxEntries</code>: hard cap on number of matches returned.</li>
</ul>
<h3>book.inject(text, opts?)</h3>
<div class="sig" data-codeid="sig6"></div>
<p>Convenience wrapper that runs <code>scan</code> and formats the result as a string ready to inject into an LLM prompt.</p>
<ul>
<li><code>format</code>: <code>'list'</code> (default; bulleted), <code>'paragraph'</code> (double-newline separated), <code>'plain'</code> (no separators or header), or a function <code>(matches) =&gt; string</code>.</li>
<li><code>header</code>: prefix line for the output. Default <code>\[Lore\]</code>.</li>
</ul>
<pre data-codeid="code7"></pre>
<h2>Token budgeting</h2>
<p>Token counts are estimated as <code>content.length / 4</code> by default. Pass <code>charsPerToken</code> to <code>create()</code> to tune for your model. This is a rough heuristic -- accurate enough for budgeting decisions, not a substitute for the real tokenizer if you need exact counts.</p>
<pre data-codeid="code8"></pre>
<h2>Persistence</h2>
<p>Pass a <code>storage</code> object (anything with <code>getItem</code> / <code>setItem</code>) and a <code>storageKey</code> to enable automatic persistence:</p>
<pre data-codeid="code9"></pre>
<p>If <code>entries</code> seed is provided alongside <code>storage</code>, the persisted data takes precedence when present.</p>
<h2>Match modes</h2>
<p><strong>Word mode</strong> (default): matches the key only at word boundaries. <code>dragon</code> matches "a dragon flew" but not "dragonfly was nearby". Punctuation and whitespace count as boundaries.</p>
<p><strong>Substring mode</strong>: matches the key as a literal substring anywhere. <code>dragon</code> would match "dragonfly". Useful when your keys are partial words or contain special characters that wouldn't match cleanly as words.</p>

<h2>Sort order</h2><div class="note">
Entries with the same priority are sorted by <em>position in the input text</em>, not insertion order. If two entries both have priority 100 and one is mentioned earlier in the message, that one comes first. This is usually what you want for context relevance.

<script>
(function(){
'use strict';

if (!window.weld) window.weld = {};
if (window.weld.lorebook) return; // already loaded

// -----------------------------------------------------------------
// Token estimation -- rough char/N ratio
// -----------------------------------------------------------------
function estimateTokens(text, charsPerToken) {
  if (!text) return 0;
  return Math.ceil(String(text).length / (charsPerToken || 4));
}

// -----------------------------------------------------------------
// Match helpers
// -----------------------------------------------------------------
function escapeRegex(s) {
  return String(s).replace(/[.*+?^${}()|[\]\\]/g, '\\$&');
}

function buildKeyMatcher(key, opts) {
  var caseSensitive = opts.caseSensitive === true;
  var matchMode = opts.matchMode || 'word';
  var flags = caseSensitive ? 'g' : 'gi';
  var escaped = escapeRegex(key);
  if (matchMode === 'substring') {
    return new RegExp(escaped, flags);
  }
  // word: match whole word with non-word boundaries
  return new RegExp('(^|[^\\w])(' + escaped + ')(?=[^\\w]|$)', flags);
}

function uid(prefix) {
  return (prefix || 'lb') + '_' + Date.now().toString(36) + Math.random().toString(36).slice(2, 6);
}

// -----------------------------------------------------------------
// Lorebook instance factory
// -----------------------------------------------------------------
function createLorebook(opts) {
  opts = opts || {};
  var entries = [];
  var charsPerToken = opts.charsPerToken || 4;
  var storage = opts.storage;
  var storageKey = opts.storageKey || 'weld-lorebook';

  if (storage === undefined && typeof localStorage !== 'undefined') {
    storage = null; // explicit opt-in for storage
  }

  // Seed entries
  if (Array.isArray(opts.entries)) {
    for (var i = 0; i < opts.entries.length; i++) {
      addInternal(opts.entries[i]);
    }
  }

  // Load from storage if provided
  if (storage) {
    try {
      var raw = storage.getItem(storageKey);
      if (raw) {
        var loaded = JSON.parse(raw);
        if (Array.isArray(loaded)) {
          entries = []; // replace seed
          for (var j = 0; j < loaded.length; j++) {
            addInternal(loaded[j]);
          }
        }
      }
    } catch (e) { /* swallow */ }
  }

  function save() {
    if (!storage) return;
    try { storage.setItem(storageKey, JSON.stringify(entries)); }
    catch (e) { /* swallow */ }
  }

  function normalizeEntry(input) {
    if (!input || typeof input !== 'object') {
      throw new Error('lorebook entry must be an object');
    }
    var keys = input.keys;
    if (typeof keys === 'string') keys = [keys];
    if (!Array.isArray(keys) || keys.length === 0) {
      throw new Error('lorebook entry must have non-empty keys');
    }
    if (typeof input.content !== 'string' || !input.content) {
      throw new Error('lorebook entry must have non-empty content');
    }
    return {
      id: input.id || uid('lb'),
      keys: keys.map(String),
      content: String(input.content),
      priority: typeof input.priority === 'number' ? input.priority : 0,
      enabled: input.enabled !== false,
      matchMode: input.matchMode || 'word',
      caseSensitive: input.caseSensitive === true,
      label: input.label || '',
      metadata: input.metadata || null
    };
  }

  function addInternal(input) {
    var entry = normalizeEntry(input);
    entries.push(entry);
    return entry;
  }

  function add(input) {
    var entry = addInternal(input);
    save();
    return entry.id;
  }

  function update(id, patch) {
    var idx = entries.findIndex(function(e) { return e.id === id; });
    if (idx === -1) return false;
    var merged = Object.assign({}, entries[idx], patch);
    entries[idx] = normalizeEntry(merged);
    save();
    return true;
  }

  function remove(id) {
    var idx = entries.findIndex(function(e) { return e.id === id; });
    if (idx === -1) return false;
    entries.splice(idx, 1);
    save();
    return true;
  }

  function get(id) {
    var found = entries.find(function(e) { return e.id === id; });
    return found ? Object.assign({}, found) : null;
  }

  function getAll() {
    return entries.map(function(e) { return Object.assign({}, e); });
  }

  function clear() {
    entries = [];
    save();
  }

  // -------- SCAN --------
  function scan(text, scanOpts) {
    scanOpts = scanOpts || {};
    var maxEntries = scanOpts.maxEntries == null ? Infinity : scanOpts.maxEntries;
    var maxTokens = scanOpts.maxTokens == null ? Infinity : scanOpts.maxTokens;

    if (typeof text !== 'string' || !text) return [];

    var matches = [];
    for (var i = 0; i < entries.length; i++) {
      var entry = entries[i];
      if (!entry.enabled) continue;
      for (var j = 0; j < entry.keys.length; j++) {
        var key = entry.keys[j];
        var re = buildKeyMatcher(key, {
          matchMode: entry.matchMode,
          caseSensitive: entry.caseSensitive
        });
        var m = re.exec(text);
        if (m) {
          matches.push({
            entry: Object.assign({}, entry),
            matchedKey: key,
            position: m.index
          });
          break; // one match per entry suffices
        }
      }
    }

    // Sort by priority descending, then by position ascending
    matches.sort(function(a, b) {
      if (b.entry.priority !== a.entry.priority) return b.entry.priority - a.entry.priority;
      return a.position - b.position;
    });

    // Enforce budgets
    var out = [];
    var tokens = 0;
    for (var k = 0; k < matches.length && out.length < maxEntries; k++) {
      var t = estimateTokens(matches[k].entry.content, charsPerToken);
      if (tokens + t > maxTokens) continue;
      tokens += t;
      out.push(matches[k]);
    }
    return out;
  }

  function inject(text, injectOpts) {
    injectOpts = injectOpts || {};
    var format = injectOpts.format || 'list';
    var header = injectOpts.header || ('[' + 'Lore' + ']');
    var matches = scan(text, injectOpts);
    if (matches.length === 0) return '';

    if (typeof format === 'function') {
      return format(matches);
    }
    var lines = [header];
    if (format === 'list') {
      for (var i = 0; i < matches.length; i++) {
        lines.push('- ' + matches[i].entry.content);
      }
    } else if (format === 'paragraph') {
      lines.push(matches.map(function(m) { return m.entry.content; }).join('\n\n'));
    } else if (format === 'plain') {
      lines.length = 0;
      for (var j = 0; j < matches.length; j++) lines.push(matches[j].entry.content);
    }
    return lines.join('\n');
  }

  function estimate(text) {
    return estimateTokens(text, charsPerToken);
  }

  return {
    add: add,
    update: update,
    remove: remove,
    get: get,
    getAll: getAll,
    clear: clear,
    scan: scan,
    inject: inject,
    estimateTokens: estimate,
    size: function() { return entries.length; }
  };
}

window.weld.lorebook = {
  create: createLorebook,
  estimateTokens: estimateTokens,
  version: '1.0.0'
};

})();

</script>

  ## <center> *How is this different from the* [`goto-plugin`](/goto-plugin) *?* </center>
  \s
  This plugin is very similar to the `goto-plugin` (it has the same code but with some modifications). It also still has the functionality of the normal `goto-plugin` but with other new features.
  
  * **Appending 'a'** - allows the user to append/add the 'goto' item into the target id (or self), but it **will not delete** the contents of the *starting* container (container of the `goto` text clicked).
  
  * **Moving 'm'** - allows the user to append/add the 'goto' item into the target id (or self), but it **will delete** the contents of the *starting* container (container of the `goto` text clicked).
  
  * **Replacing 'r'** - allows the user to replace the contents of the target id (or self) with the the 'goto' item, but it **will not delete** the contents of the *starting* container (container of the `goto` text clicked).
  
  * **Going 'g'** - allows the user to replace the contents of the target id (or self) with the the 'goto' item, but it **will delete** the contents of the *starting* container (container of the `goto` text clicked).
  \s
  [This post](https://www.reddit.com/r/perchance/comments/13p70fj/generate_further_info_by_clicking_text/)  is the inspiration of this modification of the `goto-plugin` (shout out to u/hillermylife). [Here](/vghbzpr9z2) is the prototype of the plugin before this page.

  ## <center> *Isn't this similar to [LINK Based Dialog System](/link-based-dialogue-system)?* </center>
  \s
  The work of **Dixiklo#7514** (on [Discord](https://discord.gg/43qAQEVV9a)) is a *phenomenal work of art*.
  \s
  But the LINK Based Dialog System may be quite daunting for new users to use that system (Even I still don't know how it works... it's like ✨ MAGIC ✨).
  \s
  Thus, the purpose of this plugin is to use the existing functionality of the `goto-plugin` and add the option to **not** remove the text when going to another 'location'. 
  \s
  It does not aim to **replace** the LINK Based Dialog System, but to *offer* a new tool that may be useful to others.
  ## <center> How to Use </center>
  \s
  Put this in your Perchance code panel:
  \s
  ```
  goto \= \{import:go-to-plugin\}
  ```
  \s
  > Or you can replace your `goto-plugin` import with this URL which will give you the new features but not needing to change all lines of old `goto-plugin`
  >
  > You can also replace `goto` with any variable name you want.
  \s
  If you want just the normal `goto-plugin`, you can use `\[goto(location, "text")\]` just like the old `goto-plugin`.
  \s
  If you are using the `elementID` on the `goto-plugin` and don't want to change the position of the `elementID` on the function, you just need to add the `function` like so:
  ```
  \[goto(location, "text", "elementID")\]
  \s
  to
  \s
  \[go-to(location, "text", "elementID", "function")\]
  ```
  If you don't specify the function, it will throw an error. Here is an [Example Generator](https://perchance.org/az9s6slncs#edit) to show this feature.
  \s
  The syntax of the plugin is the following:
  ```
  \[goto(location, "text", "function", "elementID", style, `sep`)\]
  ```
  \s
  ### Parameters
  Parameter | Description | Default | Accepted Values
  -|-|-|-
  `location` | This is where you will link the lists/variables that contains the text of the 'location'. (*Required*) | `none` | any list or variable
  `text` | This is the text that will be displayed that the user will click to go to the 'location'. This can be a reference to a list/variable/object to dynamically display the word that will be clicked. (*Required*) | `none` | any list or variable
  `function` | This is where you specify the function of the plugin, there are four possible values: `'a'` for appending, `'m'` for moving, `'r'` for replacing, `'g'` for going. | `'g'` | `'a'`, `'g'`, `'r'`, `'m'`
  `elementID` | This is where you specify the target element ID (like `<div id\="this-is-the-id"></div>` the id is `this-is-the-id`). | `none` | any HTML tag ID
  `style` | This is where you specify CSS Styles for the plugin | `none` | any CSS
  `sep` | This is where you add the separator for `'a'` and `'m'`. Applied before the text of the item to be appended | `<br><br>` | any Text
  \s
  > Both `function` and `elementID` inputs need to be enclosed in `" "` or `' '` or `` ` ` ``. Any paramters you don't want to use you can use `null`.
